# T10 Public Shadow

> *"il sussurro di una brezza leggera"* — the public-visible artifact of every
> Shadow Token: a 16x16 4-level grayscale silhouette posted on chain, derivable
> from the encrypted face only with a zk proof.

## What it is

`shadowT10[shadowId]` is **256 bits of public art** packed as
`(bytes32 hi, bytes32 lo)` per shadow:

```text
hi = q0 | (q1 << 128)
lo = q2 | (q3 << 128)
```

Each `q*` is a 128-bit field whose 2-bit groups encode 64 cells of a 16x16
grid; each cell holds a value in `0..3` (4 grayscale levels). Decoding:

```python
cell_idx = by * 16 + bx                     # by, bx in 0..15
q        = quarters[cell_idx // 64]
level    = (q >> ((cell_idx % 64) * 2)) & 0x3
```

## Where it comes from

The 256 bits are not stored at mint. They are **lazy-refreshed** by anyone via:

```solidity
function setShadowT10(uint256 shadowId, bytes proof, bytes32[9] pi)
```

`proof` is a Honk proof of the `shadow_t10` Noir circuit. The contract verifies
`proof` against the registered `T10ShadowVerifier`, then writes the new
`(hi, lo)` to storage and emits `ShadowT10Updated`.

### Public inputs (9 fields)

| idx | field             | bound to                                                |
|----:|-------------------|---------------------------------------------------------|
| 0   | `shadow_id`       | `pi[0] == shadowId`                                     |
| 1   | `state_nonce`     | `pi[1] == stateNonce[shadowId]`                         |
| 2   | `prev_ct_commit`  | `pi[2] == _shadows[shadowId].c2Commit`                  |
| 3   | `boxes_packed`    | `pi[3] == boxesPackedOf[shadowId]`                      |
| 4   | `poses_hash`      | `pi[4] == _posesHash(shadowId)`                         |
| 5   | `shadow_q0`       | quartet 0..63                                           |
| 6   | `shadow_q1`       | quartet 64..127                                         |
| 7   | `shadow_q2`       | quartet 128..191                                        |
| 8   | `shadow_q3`       | quartet 192..255                                        |

The five chain-binding asserts make stale fixtures impossible to land:

* The proof must be for **this** shadow (`shadow_id`).
* It must be for the shadow's **current** mutation count (`state_nonce`),
  which bumps on every `mintShadow` / `transferShadow` / `mutateSlot` /
  `extractSlot` / `insertFeature` / `removeFeature`.
* It must reference the actually-stored ciphertext (`prev_ct_commit`),
  the immutable mint geometry (`boxes_packed`), and the **current** manifest
  pose set (`poses_hash`).

## How the bytes are computed (in-circuit)

The Noir circuit `circuits/shadow_t10/src/main.nr` does, in order:

1. **Plaintext binding.** Decrypt the witness `shadow_plaintext: [Field; 249]`
   under `shadow_prev_k` (Poseidon2 keystream); assert
   `sponge_249(shadow_ct) == prev_ct_commit`.
2. **Pose binding.** Assert `sponge_18([poses[0..16], 0, 0]) == poses_hash`.
   Each pose decodes via `assert_pose_decode` into `(cx, cy, scale_q88,
   cos_q15, sin_q15)`. Slots 8..15 are forced to `pose == 0` (MVP: mutate-only).
3. **Geometry binding.** Decode `boxes_packed` into per-slot
   `(box_x, box_y, box_w, box_h)` and validate each against MAX
   `REGION_W/H`.
4. **Composite.** For each of the 8 ORIGINAL slots, paint its
   `[box_w, box_h]` sub-rect into a 48x48 RGB canvas using inverse-affine
   nearest-neighbor sampling under the slot's pose. Math is integer
   fixed-point throughout (Q15 cos/sin, Q24 inverse-scale, sign-safe
   floor division to match Python's arithmetic right-shift).
5. **Quantize.** Each 16x16 cell is the average of a 3x3 block of
   ITU-601 luma values `(77r + 150g + 29b) >> 8`, then thresholded against
   `64 / 128 / 192` to a 4-level value. Pack as 4 quartet fields and
   bind to PI.

### Pose constraints

* `scale_q88` MUST be a power of 2 in `{32, 64, 128, 256, 512, 1024, 2048,
  4096}` so the Q24 inverse `2^24 / scale` is exact and `pose_scale_inv_q24
  * scale_q88 == 2^24` can be asserted in-circuit (no division-with-remainder
  gadget needed).
* Rotation `(cos, sin)` is checked in the contract's `PoseLib.requireSane`
  (`cos^2 + sin^2 == 2^30 ± tol`). The circuit consumes them as i32.

## Sponge_18 padding (why 18 not 16)

The on-chain `Poseidon2YulSponge` rejects calldata whose length is not a
multiple of 96 bytes (3 BN254 fields). The chain therefore pads the 16
manifest poses with two zero fields and absorbs 18. The circuit and Python
reference do exactly the same thing so all three implementations agree:

```python
# Python (tools/t10.py + build_shadow_t10_fixture.py)
poses_hash = poseidon2_sponge(list(poses) + [0, 0])

# Noir (circuits/shadow_t10/src/main.nr)
let mut padded_poses: [Field; 18] = [0; 18];
for i in 0..16 { padded_poses[i] = poses[i]; }
assert(sponge_18(padded_poses) == poses_hash);

// Solidity (contracts/src/ShadowToken.sol)
function _posesHash(uint256 shadowId) internal view returns (bytes32) {
    bytes memory buf = new bytes(18 * 32);                // 16 poses + 2 zero pads
    for (uint256 i = 0; i < 16; i++) { /* mstore pose i */ }
    // staticcall yulSponge with 576 bytes
}
```

## Floor-vs-truncate division

Noir's `i64 /` truncates toward zero. Python's `>>` floors. The composite's
nearest-neighbor rounding `(sy_2 + 1) / 2` and `(sy_q31 + 2^30) / 2^31`
must use **floor** semantics to be geometrically correct (see `inv_affine_for_slot`
derivation in the circuit). The circuit explicitly implements `floor_div_2`
and `floor_div_2pow31` rather than relying on `/`:

```noir
fn floor_div_2(x: i64) -> i64 {
    if x >= 0 { x / 2 } else { -(((-x) + 1) / 2) }
}
fn floor_div_2pow31(x: i64) -> i64 {
    if x >= 0 { x / 2147483648 }
    else { -(((-x) + 2147483647) / 2147483648) }
}
```

Without these helpers, a slot under non-identity scale will paint pixels
just outside its geometric extent that should have been clipped — visible
as a 1-pixel halo of bleed in the public T10 grid.

## Limitations (MVP)

* **Mutate-only.** Slots 8..15 are required to be `EMPTY` (`pose == 0`).
  Inserts are deferred to v2 (the witness-side composite already supports
  16 slots; the v2 contract change is removing the `for k in 8..16
  { assert(poses[k] == 0); }` constraint and adding inserted-feature ct
  commits to the PI).
* **Power-of-2 scale only.** Scales must be in `{1/8, 1/4, 1/2, 1, 2, 4, 8,
  16}` (Q8.8 values 32..4096).
* **Lazy refresh.** `shadowT10` is briefly stale between a state-change tx
  and the corresponding `setShadowT10` tx. Indexers should subscribe to
  both `SlotMutated` / `ShadowTransferred` etc. and `ShadowT10Updated`
  to reconcile.
